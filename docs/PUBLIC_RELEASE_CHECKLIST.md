# Public Release Checklist

Use this checklist before uploading the repository to GitHub and citing it in a thesis or paper.

- Confirm that the selected project license matches your intended release terms.
- Replace placeholder contact details with real public contact emails.
- Replace placeholder repository and documentation references with the final public URLs.
- Confirm whether `SXdata` may be redistributed.
- If `SXdata` may not be redistributed, state that clearly in the manuscript and repository.
- Install dependencies and verify the environment with a documented smoke test.
- Optionally add a paper, thesis, or DOI reference to `CITATION.cff`.
